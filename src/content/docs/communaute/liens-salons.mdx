<p class="doc-lead">Le module <strong>Liens de salons</strong> permet de connecter des salons textuels afin de relayer automatiquement les messages qui y sont envoyés. C'est l'outil idéal pour connecter deux communautés ou créer un canal de discussion partagé entre plusieurs serveurs.</p>

:::note[Disponibilité]
Cette fonctionnalité est accessible depuis la section **Cross-Serveur → Liens de salons** de votre dashboard. Vous devez être administrateur du serveur pour créer ou modifier ces liens.
:::

---

## Fonctionnement du relais

Lorsque deux salons sont liés, Kotbo écoute les nouveaux messages et les renvoie instantanément dans le salon connecté. Deux modes de relais sont disponibles :

*   **Webhook (Miroir)** *(Recommandé)* : Le bot utilise un webhook Discord pour envoyer le message avec le pseudo et la photo de profil réels de l'auteur original. Le rendu est propre et naturel.
*   **Embed** : Le message est encapsulé dans un encart (embed) stylisé envoyé au nom de Kotbo, mentionnant le serveur et l'auteur d'origine.

Vous pouvez configurer la direction du lien :
*   **Bidirectionnel** : Les messages du salon A vont vers B, et inversement.
*   **Unidirectionnel** : Les messages vont uniquement du salon source (A) vers le salon cible (B).

---

## Cas 1 : Lier deux salons sur le même serveur

Vous pouvez lier deux salons distincts au sein de votre propre serveur (par exemple, pour dupliquer des annonces ou relayer des discussions d'un salon privé vers un salon public).

### Étapes de configuration
1. Ouvrez le dashboard de votre serveur et accédez à **Cross-Serveur → Liens de salons**.
2. Cliquez sur le bouton **Créer un lien** en haut à droite.
3. Remplissez le formulaire :
   *   **Salon sur ce serveur** : Choisissez le premier salon (source).
   *   **Serveur cible** : Sélectionnez votre propre serveur dans la liste.
   *   **Salon sur le serveur cible** : Sélectionnez le deuxième salon (la liste exclut automatiquement le salon source déjà choisi).
   *   **Direction** et **Mode de relay** : Ajustez selon vos besoins (ex: *Bidirectionnel* et *Webhook*).
4. Cliquez sur **Créer le lien**. La liaison est instantanée !

*Alternative Discord :* Vous pouvez également créer cette liaison directement sur Discord avec la commande slash :
```text
/link salon salon-source:#salon-A salon-cible:#salon-B direction:Bidirectionnel mode:Webhook
```

## Cas 2 : Lier deux salons sur des serveurs différents

Pour connecter deux serveurs Discord distincts, le bot Kotbo doit être installé sur les deux serveurs. Deux méthodes s'offrent à vous selon vos droits sur le serveur distant.

### Méthode A : Liaison directe (si vous êtes administrateur des deux serveurs)
Si vous possédez les droits d'administration sur les deux serveurs, vous pouvez établir le lien directement depuis votre dashboard.

1. Allez dans **Cross-Serveur → Liens de salons** sur le serveur source.
2. Cliquez sur **Créer un lien**.
3. Sélectionnez le salon source local.
4. Dans **Serveur cible**, sélectionnez le serveur distant (celui-ci apparaît dans la liste si vous y êtes administrateur et que le bot y est présent).
5. Sélectionnez le salon cible dans la liste.
6. Ajustez vos options et cliquez sur **Créer le lien**.

*Alternative Discord :* 
```text
/link direct salon-source:#salon-local serveur-cible:ID_SERVEUR_CIBLE salon-cible:ID_SALON_CIBLE
```

### Méthode B : Liaison par invitation (si le serveur cible appartient à un autre propriétaire)
Si vous souhaitez lier votre salon à celui d'un autre serveur que vous ne gérez pas, vous devez générer un code d'invitation que l'autre administrateur devra valider. Dans ce scénario, le **Serveur A** désigne le serveur depuis lequel vous initiez la configuration et créez l'invitation, et le **Serveur B** désigne le serveur cible qui recevra l'invitation.

#### Étape 1 : Générer l'invitation (Serveur A — Serveur d'initiation)
1. Sur le serveur A, allez dans **Cross-Serveur → Liens de salons** et cliquez sur **Créer un lien**.
2. Sélectionnez votre salon local.
3. Activez l'option **Générer une invitation Discord** (cela permet également de créer une invitation d'accès si les membres veulent rejoindre l'autre serveur).
4. Sélectionnez la direction et le mode de relais souhaités.
5. Cliquez sur **Créer le lien**.
6. Une boîte verte s'affiche avec un code de liaison (ex: `A1B2C3D4`) ainsi qu'un lien d'invitation Discord si vous avez coché l'option. **Copiez ce code** et transmettez-le à l'administrateur du serveur distant.

*Alternative Discord :* 
```text
/link invite salon:#salon-local direction:Bidirectionnel mode:Webhook invitation-serveur:Vrai
```

#### Étape 2 : Accepter l'invitation (Serveur B — Serveur cible)
L'administrateur du serveur B doit valider le lien depuis son propre serveur Discord en exécutant la commande slash suivante :
```text
/link accept code:CODE_D_INVITATION salon:#salon-local-cible
```
Une fois validé, Kotbo crée automatiquement les webhooks nécessaires et connecte les deux salons.

## Personnaliser le contenu relayé

Par défaut, Kotbo synchronise les textes et les images. Vous pouvez affiner ce comportement pour chaque lien actif.

1. Dans la liste de vos liens sous **Cross-Serveur → Liens de salons**, repérez le lien concerné et cliquez sur l'icône d'engrenage (**Configurer**).
2. Activez ou désactivez les interrupteurs selon vos préférences :
   *   **Texte** : Relaye les messages écrits classiques.
   *   **Images** : Relaye les images et fichiers joints.
   *   **Embeds** : Relaye les fenêtres d'intégration générées par des liens externes ou des bots.
   *   **Réactions** : Relaye les émojis ajoutés en réaction aux messages.
   *   **Éditions** : Si un utilisateur modifie son message d'un côté, la modification est répercutée de l'autre.
   *   **Suppressions** : Si un message d'origine est supprimé, sa copie dans l'autre salon l'est également.
3. Cliquez sur **Sauvegarder**.

:::warning[Permissions Discord requises]
Pour que le mode **Webhook (Miroir)** et le relais des pièces jointes fonctionnent correctement, veillez à ce que le rôle de Kotbo dispose des permissions **Gérer les Webhooks** et **Ajouter des fichiers** dans les salons concernés sur les deux serveurs.
:::

## Supprimer une liaison

Pour rompre un lien, retournez dans **Cross-Serveur → Liens de salons**, cliquez sur l'icône de corbeille rouge (**Supprimer**) à côté du lien et confirmez. Le relais s'arrêtera immédiatement.

Si l'option **Mettre à jour la description des salons** était active, le bot supprimera automatiquement les liens d'invitation insérés dans le sujet (topic) des salons Discord concernés.

*Alternative Discord :*
```text
/link remove id:ID_DU_LIEN
```
*(Utilisez `/link list` pour trouver l'identifiant unique du lien à supprimer)*.
